Poker AI - Sealed-State Multi-Agent Simulation

Sealed-state multi-agent simulation where 4 LLM-powered agents play Texas Hold'em with fully isolated private memory and adversarial reasoning.

4 Agents with isolated private state and independent per-agent memory
Top 8 Out of 130 builders at NYC Tech Week, co-hosted by NVIDIA x a16z
0 Runtime LLM calls during demo because outputs were pre-generated offline

What It Solves

Most multi-agent demos give all agents shared context. Here, each agent operates with sealed private state, knows only its own hole cards, remembers its own history, and builds its own read on the table.

Why It Matters

Poker is the demo surface. The architecture generalizes to sealed-information multi-agent problems like M&A negotiation, litigation strategy, and competitive market simulation.

What I Built

  • Designed the agentic graph and backend skeleton with 4 distinct behavioral profiles and bluff rates from 0-40%
  • Built an offline OpenRouter pipeline using NVIDIA Nemotron 70B to produce a static JSON event log for React playback
  • Generated ElevenLabs MP3 narration per agent action with subtitle fallback if audio fails

Reveal Layer

  • Post-hand reveal modal surfaces each agent's full reasoning only after the hand concludes
  • Reasoning is tagged with BLUFF, TRAP, HERO_CALL, TILT, and OPTIMAL_FOLD labels
  • No agent sees another agent's reasoning before the post-hand reveal
